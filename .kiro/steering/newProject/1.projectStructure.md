# Estructura de Proyecto Base

Esta es la estructura estándar para todos los proyectos. Stack: Node.js + Express, JS vanilla (front), MySQL + Sequelize.

## Estructura de Carpetas

```
/proyecto
├── database/
│   ├── config/          # Configuración de Sequelize (conexión a DB)
│   ├── migrations/      # Migraciones de Sequelize
│   ├── models/          # Modelos de Sequelize
│   └── seeders/         # Seeders de datos iniciales
│
├── public/              # Archivos estáticos servidos por Express
│   ├── css/             # Hojas de estilo
│   │   └── custom/      # Estilos propios del proyecto
│   ├── files/           # Archivos (uploads u otros)
│   ├── fonts/           # Fuentes tipográficas
│   ├── images/          # Imágenes
│   └── js/              # JavaScript del frontend
│
├── scripts/             # Scripts de automatización
│
├── sqlScriptsAndData/   # Scripts SQL y datos auxiliares
│
├── src/                 # Código fuente del backend
│   ├── controllers/     # Controladores (lógica de request/response)
│   ├── data/            # Datos estáticos o auxiliares
│   ├── dbQueries/       # Queries a base de datos
│   ├── middlewares/     # Middlewares de Express
│   ├── routes/          # Definición de rutas
│   ├── utils/           # Funciones utilitarias
│   ├── validations/     # Validaciones de datos
│   └── views/           # Vistas (templates)
│       ├── login/       # Vistas de login
│       └── partials/    # Parciales reutilizables
│
├── .gitignore
├── .sequelizerc         # Configuración de paths de Sequelize CLI
├── app.js               # Entry point de la aplicación
├── package.json
├── package-lock.json
└── readme.md
```

## Reglas de Estructura

- Todo el código fuente del backend va en `src/`
- Los archivos estáticos del frontend van en `public/`
- La configuración de base de datos va en `database/`
- Nunca mezclar lógica de backend en `public/` ni archivos estáticos en `src/`
- Las vistas parciales reutilizables van en `src/views/partials/`
